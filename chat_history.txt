🔧 What changed
FoodDonationSerializer

Added status to fields.
read_only_fields = ['created_at', 'status'].
Added validate_hours_until_expiry (must be > 0).
FoodDonationListCreateView

perform_create now:
enforces request.user.role == 'donor'
saves donor=request.user
raises PermissionDenied for non-donors.
New FoodDonationStatusUpdateView

POST /<pk>/pickup/:
only role == volunteer
status transition available -> picked_up
POST /<pk>/complete/:
only role == receiver
status transition picked_up -> completed
returns 400 for invalid action or wrong status.
urls.py

route '' → FoodDonationListCreateView
route '<int:pk>/<str:action>/'int:pk/str:action/' → FoodDonationStatusUpdateView
tests.py

new FoodDonationPermissionsTests:
donor create allowed, volunteer create denied
volunteer pickup + receiver complete status transitions
